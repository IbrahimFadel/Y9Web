Inquiry & Analysis


Explanation & Justification

Explanation

Today, data is being collected all the time. However it is not being used or presented effectively. With organizations such as the World Health Organization collecting data on many things such as Life Expectancy, it is important that we use it effectively. The problem is that this is all valuable information, but as it stands now, as tabular data it is ineffective and unuseful. This website will take this tabular data from the WHO’s website in the form of JavaScript Object Notation, or JSON for short and represent it in a beautiful way.

Justification

This process explained above is important to undergo for many reasons. Firstly, tabular data is next to useless. Humans were built to interpret visual data, not mathematical. So by employing the computer’s strengths(computing) we can take tabular data and spit it out in the form of visually appealing, easy to understand graphs and charts. This makes it easy to see trends, and draw conclusions. This representation of the data can help students doing research in these areas. Without this website, the data presented by the WHO is next to useless. The justification for the website is that it makes the data useful for users.

Target Audience / Client

The target audience of my product is highschool students. These students in particular are going to be taking humanities courses that look at things like life expectancy, and analyze the statistics they find. My product could be a big help for these students. While the product is still useful to adults, the visual representation is more suited to teenagers. This product is right at the threshold of informative and easy to understand. This is why the target audience of my product is highschool students.


Primary Research / Skills

Skills Needed

The skills i will need to know are varying and in some cases, complicated. I must first be able to download the JSON and write a javascript program to read it, and get the raw data from it. Next I must use the raw data, and write some javascript to display it on an html canvas. The website shall have many different functionalities such as reversing the order of the list, ordering based on alphabetical, the data values, and finally displaying data from different years. These functions will all require javascript. However what’s often overlooked is the backbone of the website which is the HTML. This is the make or break of the website. Without it, the user cannot navigate click any buttons(because there are none), etc. Much of my time will be spent making the HTML. Finally an immense amount of CSS will be used. The user experience is extremely important. Without good styling of a website, the user is almost guaranteed to leave, as such, I will need profound skills in CSS.

Priority

The number one priority will definitely be the javascript code. This is for many reasons, but most importantly, it is the hardest code to write out of the three needed for this project. It will be the most time consuming, and also carries out the core functionality. Without the javascript, I would have to copy down each data point into a table by hand which is unrealistic. The next priority will be the html. This could arguably be the top priority, but since the javascript could display the data on the console without html, html will be second in priority. Without the html, we have no website. Merely a command line program which is still useful, but not in accordance with our goal which is to present an easy to use, read and understand representation of data. With html, we have access to the canvas tag which will allow us to display the data graphically without the use of extensions such as flash. Html also allows for buttons, and links, text and much more. Without it, this website could not exist which is why it is the second priority. Last but not least is CSS. While this is crucial to the website, it does not contribute to functionality, it ensures that the user stay on the website for the longest amount of time. With an ugly website, there is zero chance that a user will stay on for long which is why it is still a priority, but the last one.

Design Brief

After researching many products that were presenting data both tabularly and visually, I have decided to create my own product that takes some of the best aspects of each product I saw, and will try to exclude the negative aspects.




My product will:


  • Show life expectancy of countries visually
  • Be able to show data from selected years
  • Be represented in different forms(line graph, bar graph, possibly map)
  • Be aesthetically pleasing
  • Load relatively quickly
  • Have Customization(reverse order, change colours, etc.)
  • Help educate on the topic
  • Have a good user experience
  • Be intuitive
  • Have every button work properly(everything has a function)
  • Have links to WHO’s data
  • Provide highschool students easily accessible/understandable data

My research is valid because:

  • All had same criticisms
  • Websites looked at are credible

This is relevant because:


  • It solves the problem of tabular data being incomprehensible to most people
